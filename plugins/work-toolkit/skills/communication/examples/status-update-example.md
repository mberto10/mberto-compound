# Status Update Example

## Real-World Example

```
Betreff: Status Update: RAG Pipeline Optimierung - KW51

Hallo zusammen,

hier ein kurzes Update zum aktuellen Stand der RAG Pipeline Optimierung:

**Erledigt diese Woche:**
- Embedding Model auf German BERT umgestellt (15% bessere Retrieval Accuracy)
- Chunking Strategy optimiert (512 → 256 tokens mit overlap)
- Performance Tests abgeschlossen: 95th percentile jetzt bei 1.8s (vorher 3.2s)

**In Arbeit:**
- A/B Test Setup für neuen Ranking Algorithmus
- Integration mit bestehendem Feedback Loop

**Blocker:**
- Warten auf Staging Environment Update (DevOps Ticket: INFRA-234)

**Nächste Schritte:**
- A/B Test Launch geplant für KW52
- Stakeholder Demo am 20.12. um 14:00

Bei Fragen meldet euch gerne.

Beste Grüße
Max
```

## Key Elements

1. **Clear subject** with project name and KW
2. **Structured sections** for easy scanning
3. **Specific metrics** where available
4. **Honest about blockers** with reference to tracking
5. **Concrete next steps** with dates
6. **Open for questions**

## Variations

### To Management (Higher-Level)

Focus on outcomes and business impact:
- "15% bessere Accuracy" → "Nutzer finden relevante Dokumente schneller"
- Less technical detail
- More emphasis on timeline and risks

### To Technical Team

More detail is fine:
- Include ticket numbers
- Technical decisions and rationale
- Links to PRs/documentation
